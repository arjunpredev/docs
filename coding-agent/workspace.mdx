## Slash commands

Type `/` in the chat composer to open the palette:

<Frame caption="The slash palette: build controls without leaving the composer">
  <img src="/images/product/slash-commands.jpg" alt="The slash palette: build controls without leaving the composer" />
</Frame>

| Command | What it does |
|---------|-------------|
| `/pro` | Toggle Pro Mode â€” pin every phase to the Pro model |
| `/model` | Choose models per phase (chat / research / coding / acceptance) |
| `/balance` | Show your remaining credit balance |
| `/effort` | Set the sprint effort level (auto / low / medium / high) |
| `/sprint <feature>` | Launch a custom sprint in a new session |
| `/fork <prompt>` | Spin a prompt off into a new isolated session |
| `/reverse` | Map an existing codebase with reverse engineering |
| `/kanban` | Jump to the Kanban board |
| `/roadmap` | Jump to the Roadmap / Gantt |
| `/arch` | Jump to the Architecture graph |

A few more (`/skills`, `/mcp`, `/integrations`, `/share`, `/clear`, `/help`) appear in the palette marked "soon" â€” skills, MCP servers, and integrations are managed from the Integrations page until they go live in the composer.

Two commands are two-level: `/effort ` lists the levels inline, and `/model ` walks you through phase, then model â€” all without leaving the composer.

<Note>
  The same workspace concepts â€” the agent chat, slash commands, effort and model settings â€” work identically in the pre.dev CLI. See the [CLI overview](/cli/overview).
</Note>
